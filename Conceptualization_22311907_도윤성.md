## 1. Business purpose
### 1.1. Project background

'2024년 한국인의 수면 실태' 보고서에 따르면, 한국인의 평균 수면 시간은 6시간 58분으로 OECD 평균보다 18% 부족합니다. 수면 장애 및 불면증 진료 환자는 2010년 대비 약 140% 급증(약 67만 8,000명)하여 수면 건강이 개인의 문제를 넘어 국가적 차원의 관리 대상으로 부상했습니다. 보건복지부는 수면을 잘 취하기 위해선 일정한 생체리듬 유지를 강조하고 있습니다. 이는 단순히 수면량을 늘리는 것이 아닌 잠자리에 들고 일어나는 시간을 일정하게 유지하여 뇌 기능 회복과 심리적 안정에 결정적인 역할을 하기 때문입니다. 일반적으로 권장하는 적정 수면 시간이 있지만 유전적, 환경적 요인에 따라 적정 수면 시간은 모두 다를 수 밖에 없습니다. 타인이 정해준 평균에 맞추려는 시도는 오히려 사용자에게 압박감을 주어 수면의 질을 떨어뜨리는 결과를 초래하기도 합니다.

+ **나만의 Golden Time 추적**: 본 시스템은 타인이 정의한 평균 수치가 아닌, 사용자 본인이 직접 입력한 주관적 컨디션 데이터를 기반으로 분석을 수행합니다.
+ **데이터로 증명하는 규칙성**: 추상적인 권고를 넘어, 데이터 기반의 피드백을 제공함으로써 사용자가 스스로 건강한 생체리듬을 찾아가도록 돕는 것입니다.


### 1.2. Goal
사용자가 자신의 수면 패턴과 컨디션의 상관관계를 파악하여 최적의 수면 효율을 찾도록 돕는 시각화 시스템을 개발합니다. 


### 1.3. Target Market
수면 장애, 불면증을 가진 환자
본인의 적정 수면 시간을 알고 싶은 사용자

## 4. Concept of operation

### 1) 수면 데이터 기록

| Purpose | 사용자의 매일 수면 패턴과 주관족 컨디션 데이터 수집 |
| :--- | :--- |
| Approach | 사용자가 앱의 입력 폼을 통해 취침/기상 시각과 점수를 입력하면 로컬DB에 저장 |
| Dynamics | 사용자가 기록을 원하는 시점에 발생 |
| Goals | 개인별 수면 분석을 위한 데이터 확보 |

### 2) 수면 리듬 시각화

| Purpose | 생체리듬의 일정성을 직관적으로 파악할 수 있는 시간적 도구 제공 |
| :--- | :--- |
| Approach | DB의 시간 데이터를 가로형 막대 그래프로 시각화하여 화면에 출력 |
| Dynamics | 사용자가 메인 대시보드나 분석 탭에 접속할 때 실시간으로 생성 |
| Goals | 사용자가 일정한 생체리듬의 유지 여부를 스스로 인지하도록 함 |

### 3) 개인 맞춤형 적정 수면 분석

| Purpose | 사용자 본인에게 최적화된 수면 시간 도출 |
| :--- | :--- |
| Approach | 컨디션 점수가 높은 날의 수면 시간을 필터링하여 평균값 및 빈도 분석 로직 실행 |
| Dynamics | 데이터가 일정량 누적된 후 사용자가 분석 리포트를 요청할 때 실행 |
| Goals | 사용자만의 적정 수면 시간을 제시해 수면의 질을 개선할 수 있는 근거 제 |

### 4) 시스템 유지보수 및 데이터 관리

| Purpose | 앱의 안정적 구동 및 사용자 데이터의 무결성 유지 |
| :--- | :--- |
| Approach | 관리자가 시스템 로그를 모니터링하고 필요 시 DB의 인덱싱 최적화 및 버전 업데이트 수행 |
| Dynamics | 정기적인 점검 주기 또는 시스템 오류 로그가 감지됐을 때 관리자에 의해 실행 |
| Goals | 장기적인 서비스 이용을 위한 시스템 신뢰성 및 성능 유지 |
